# Documentation Quality Checklist

## Purpose

Define the quality bar for the documentation set.

## Checklist

- no obsolete docs remain in the active docs root except `COPILOT_INSTRUCTIONS.md`
- no contradictory auth statements
- no active organization-centric product flow in source-of-truth docs
- Foundry IQ is explained clearly
- Azure AI Foundry configuration is documented
- local run instructions are present
- judging rubric mapping exists
- demo flow is clear
- future work is separated from current scope
- no secrets or credentials are included
- synthetic data policy is documented

## Future Enhancements

Future documentation reviews can add link validation and consistency automation.
