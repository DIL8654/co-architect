# Azure Well-Architected Summary

## Purpose

Cloud architecture guidance for Microsoft Azure workloads.

## Key Guidance

- reliability through resilience, recovery, and continuity planning
- security through identity, network, secrets, and data protection controls
- cost optimization through right-sizing and managed service choices
- operational excellence through observability, automation, and repeatable operations
- performance efficiency through scaling, latency control, and service fit

## Use When

Use when the architecture clearly relies on Azure services or Azure-hosted platform decisions.
