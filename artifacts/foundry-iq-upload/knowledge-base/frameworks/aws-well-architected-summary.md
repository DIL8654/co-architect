# AWS Well-Architected Summary

## Purpose

Cloud architecture guidance for Amazon Web Services workloads.

## Key Guidance

- operational excellence for operations, change management, and continuous improvement
- security for identity, infrastructure protection, and data protection
- reliability for scaling, failure recovery, and change tolerance
- performance efficiency for service fit and elastic scaling
- cost optimization for waste reduction and spend visibility
- sustainability for resource-aware platform choices

## Use When

Use when the architecture is AWS-based or the team is comparing AWS patterns against another cloud option.
