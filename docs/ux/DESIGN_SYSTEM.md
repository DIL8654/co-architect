# Design System

## Purpose

Capture the visual and interaction principles for the MVP.

## Current Scope

The UI is minimal, engineering-focused, and table-first.

## Design Decisions

- workspace-centric shell
- compact header
- left navigation
- table-first information density
- minimal card usage
- clear score and finding emphasis
- dark and light theme support

## Interaction Principles

- primary actions should be clear
- navigation should not be duplicated in content
- findings and reasoning should feel inspectable
- visual noise should stay low

## Future Enhancements

- more formal component inventory
- stricter token system
