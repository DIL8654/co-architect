# Architecture Trade-Off Principles

CoArchitect trade-off reasoning should surface:

- simplicity versus scalability
- cost versus reliability
- speed of delivery versus governance
- security versus usability
- consistency versus autonomy
- build versus buy
- managed service versus self-hosted
- vendor lock-in versus platform leverage
- performance versus maintainability
- strong isolation versus operational cost

Supporting decision habits:

- capture decisions as ADRs
- make trade-offs explicit
- prefer fitness-for-purpose over ideology
- revisit decisions as constraints change
- let teams adjust priorities through visible weighting
