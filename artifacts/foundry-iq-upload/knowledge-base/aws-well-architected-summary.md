# AWS Well-Architected Summary

Source:
AWS Well-Architected Framework. Use the official AWS guidance as the canonical reference.

Summary:

- Operational Excellence covers operations, change management, and continuous improvement.
- Security covers identity, detective controls, infrastructure protection, and data protection.
- Reliability covers failure recovery, scaling, testing, and change tolerance.
- Performance Efficiency covers resource fit, monitoring, and elastic scaling.
- Cost Optimization covers avoiding waste, selecting efficient services, and measuring spend.
- Sustainability covers energy-efficient design and resource-aware choices.

CoArchitect use:

- AWS-centric architecture review
- cloud service trade-off comparison with Azure-oriented guidance
- cloud platform reasoning when architecture signals AWS usage
